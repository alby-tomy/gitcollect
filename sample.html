The problem

            
Git is flat. Your work isn't.

            
Repos belong to domains. Git doesn't know that, and it shows every time you set up a
                new machine.


            

                

                    
Without gitcollect

                    
    
                        
  • Clone 12 security repos one by one
    • 
                        
  • Forget which repos belong to a project
    • 
                        
  • Run git pull in each folder manually
    • 
                        
  • Scroll through 40 repos to find the right one
    • 
                        
  • Re-figure out the setup on every new machine
    • 
                    

                

                

                    
With gitcollect

                    
    
                        
  • gitcollect clone cybersecurity
    • 
                        
  • Collections make intent explicit and permanent
    • 
                        
  • gitcollect pull cybersecurity — all repos
    • 
                        
  • gitcollect list shows every domain at once
    • 
                        
  • Share collection.yaml in your dotfiles repo
    • 
                    

                

            

        

    


    
    

        

            
Getting started

            
Up and running in 3 minutes

            
Install once, authenticate once, then define collections to match how you actually
                think about your work.


            


                

                    
1

                    

                        
Install

                        
Download the binary for your platform and put it in your PATH. No runtime required — it is a
                            single self-contained executable.

                        

                            
                            
                            
                        

                        

                            
$ curl -L
                                https://github.com/yourusername/gitcollect/releases/latest/download/gitcollect_darwin_arm64.tar.gz
                                | tar xz
                                $ sudo mv gitcollect /usr/local/bin/
                                $ gitcollect version
                                gitcollect v1.0.0 darwin/arm64
                            

                        

                        

                            
$ curl -L
                                https://github.com/yourusername/gitcollect/releases/latest/download/gitcollect_linux_amd64.tar.gz
                                | tar xz
                                $ sudo mv gitcollect /usr/local/bin/
                                $ gitcollect version
                                gitcollect v1.0.0 linux/amd64
                            

                        

                        

                            
$ brew install
                                yourusername/tap/gitcollect
                                $ gitcollect version
                                gitcollect v1.0.0 darwin/arm64
                            

                        

                    

                


                

                    
2

                    

                        
Authenticate

                        
Run gitcollect auth and
                            enter a GitHub personal access token when prompted. The token input is hidden — it never
                            appears on screen or in shell history. Your token is stored at ~/.gitcollect/config with permissions
                            0600.

                        

                            $ gitcollect auth
                            Enter GitHub token: (hidden)
                            ✓ Authenticated as yourusername
                        

                        
Generate a token at
                            github.com/settings/tokens — requires repo scope for private
                            repos, public_repo for public only.

                    

                


                

                    
3

                    

                        
Create a collection

                        
A collection is a named group of repos under your GitHub account or organisation. Create one
                            with gitcollect init.

                        

                            $ gitcollect init cybersecurity --owner yourusername --description "Security research tools"
                            ✓ Created collection: cybersecurity
                             Location: ~/.gitcollect/collections/cybersecurity.yaml
                        

                    

                


                

                    
4

                    

                        
Add repositories

                        
Add repos by name — gitcollect resolves the rest from the GitHub API.

                        

                            $ gitcollect add cybersecurity pen-test-tools
                            ✓ Added pen-test-tools to cybersecurity
                            $ gitcollect add cybersecurity vuln-scanner
                            ✓ Added vuln-scanner to cybersecurity
                            $ gitcollect add cybersecurity ctf-writeups
                            ✓ Added ctf-writeups to cybersecurity
                        

                    

                


                

                    
5

                    

                        
Clone your collection

                        
Clone everything at once, or pick exactly what you need today.

                        

                            # Clone all repos in the collection (4 parallel by default)
                            $ gitcollect clone cybersecurity

                            # Clone only the two repos you need right now
                            $ gitcollect clone cybersecurity --pick pen-test-tools vuln-scanner

                            [1/2] Cloning pen-test-tools... done (1.2s)
                                [2/2] Cloning vuln-scanner... done (0.8s)

                                ✓ Cloned 2 repos in 1.5s
                        

                    

                


            

        

    


    
    

        

            
Commands

            
Everything you can do

            
Every command follows the same pattern: gitcollect <command> <collection> [options]


            

                

                    
Managing collections

                    

                        

                            gitcollect init <name>
                            Create a new collection
                        

                        

                            gitcollect add <collection> <repo>
                            Add a repo to a collection
                        

                        

                            gitcollect remove <collection> <repo>
                            Remove a repo from a collection
                        

                        

                            gitcollect list
                            List all your collections
                        

                        

                            gitcollect show <collection>
                            Show all repos in a collection
                        

                    

                


                

                    
Git operations

                    

                        

                            gitcollect clone <collection>
                            Clone all repos in collection
                        

                        

                            gitcollect clone <name> --pick r1 r2
                            Clone only specific repos
                        

                        

                            gitcollect clone <name> --dry-run
                            Preview without cloning
                        

                        

                            gitcollect pull <collection>
                            git pull in every repo
                        

                        

                            gitcollect status <collection>
                            git status in every repo
                        

                    

                


                

                    
Key flags

                    

                        

                            --pick repo1 repo2
                            Select repos by name
                        

                        

                            --concurrency N
                            Parallel operations (default: 4)
                        

                        

                            --dest <dir>
                            Target directory for clone
                        

                        

                            --dry-run
                            Preview without executing
                        

                        

                            --json
                            Machine-readable output
                        

                    

                


                

                    
System

                    

                        

                            gitcollect auth
                            Store GitHub / GitLab token
                        

                        

                            gitcollect version
                            Print version and platform
                        

                        

                            gitcollect completion bash|zsh|fish
                            Generate shell completion
                        

                    

                

            


            
collection.yaml — the file behind every collection

            

                # ~/.gitcollect/collections/cybersecurity.yaml
                version: "1"
                name: cybersecurity
                description: "Penetration testing and security
                    research tools"
                host: github.com
                owner: yourusername
                tags: [security, red-team]
                repos:
                - pen-test-tools
                - vuln-scanner
                - ctf-writeups
                - nmap-scripts
                
                    # Edit this file directly, or use gitcollect add/remove
                    # Commit it to your dotfiles repo to share across machines
            

        

    


    
    

        

            
Design principles

            
Built to a standard, not a deadline

            
Five properties every part of the codebase is measured against — not aspirations,
                hard constraints.


            

                

                    
Efficiency

                    
Concurrent by default

                    
Cloning and pulling run in parallel with a configurable semaphore. A collection of 10 repos
                        completes in the time of the slowest 4, not all 10 sequentially.

                

                

                    
Security

                    
Token never leaks

                    
Stored at 0600 permissions. Input is always hidden. Never logged, never printed, never passed as
                        a CLI flag that ends up in shell history.

                

                

                    
Reliability

                    
Atomic writes, partial success

                    
Collection files are written atomically — a crash never corrupts your config. If 3 of 10 clones
                        fail, the other 7 succeed and all failures are reported together.

                

                

                    
Consistency

                    
Predictable everywhere

                    
Errors go to stderr, output to stdout, so pipes work. Colours disable automatically when output
                        is not a TTY. Exit codes follow POSIX convention.

                

                

                    
Usability

                    
Errors guide, not block

                    
Every error message tells you what happened and what to do next. Typos in repo names get a
                        closest-match suggestion. The first-run experience is a single command.

                

                

                    
Zero deps

                    
Single binary

                    
No Node, no Python, no runtime. Download one file and it works. Cross-compiled for Linux, macOS
                        (Intel + Apple Silicon), and Windows from a single codebase.

                

            

        

    


    
    

        

            
Installation

            
Get the binary

            
Pick your platform. No package manager required — download, move to PATH, done.


            

                

                    
macOS — Apple Silicon (M1/M2/M3)

                    
$ curl -L
                        https://github.com/yourusername/gitcollect/releases/latest/download/gitcollect_darwin_arm64.tar.gz
                        | tar xz && sudo mv gitcollect /usr/local/bin/

                

                

                    
macOS — Intel

                    
$ curl -L
                        https://github.com/yourusername/gitcollect/releases/latest/download/gitcollect_darwin_amd64.tar.gz
                        | tar xz && sudo mv gitcollect /usr/local/bin/

                

                

                    
Linux — amd64

                    
$ curl -L
                        https://github.com/yourusername/gitcollect/releases/latest/download/gitcollect_linux_amd64.tar.gz
                        | tar xz && sudo mv gitcollect /usr/local/bin/

                

                

                    
Homebrew

                    
$ brew install yourusername/tap/gitcollect

                

                

                    
Build from source (requires Go 1.22+)

                    
$ go install
                        github.com/yourusername/gitcollect@latest

                

                

                    
Windows — download the .zip from

                    
https://github.com/yourusername/gitcollect/releases
